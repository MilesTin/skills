# 阿里拍卖捡漏助手

此技能帮助用户在阿里拍卖中寻找高性价比的特价房源（捡漏）。

## 核心工作流程

### 1. 搜索拍卖房源

调用 `search_auction_houses` MCP 工具搜索房源。

### 2. 筛选捡漏目标

根据以下条件筛选潜在的捡漏房源：

| 条件 | 说明 | 计算方法 |
|------|------|----------|
| 拍卖状态 | 正在进行中 | `status == "ing"` |
| 单价 | 每平方米低于400元 | `current_price_yuan / area < 400` |
| 剩余时间 | 小于30分钟 | `left_time_ms < 30 * 60 * 1000` |

**重要单位转换：**
- 价格：1元 = 100分，需要除以100转换为元
- 时间：剩余时间 `left_time` 单位为毫秒，30分钟 = 1800000毫秒

### 3. 获取拍卖详情

对筛选出的房源，调用 `get_auction_bid_detail` 获取详细拍卖信息（含剩余时间）。

### 4. 获取标的物描述

调用 `get_auction_item_description` 获取房源装修等信息。

### 5. 筛选装修程度

根据 `decoration` 字段筛选装修程度。合格的装修等级（从高到低）：
1. 豪华装修
2. 精装修 ✓（最低要求）
3. 普通装修
4. 毛坯（不合格）

### 6. 筛选交易类型、房屋类型和面积

根据以下字段进一步筛选：
- `transaction_type`: 流转方式必须是 **出售**
- `house_usage`: 房屋用途必须是 **普通住宅**
- `area`: 面积必须 **>= 50 平方米**

### 7. 生成新的一次新任务，删除当前任务
```
1. 筛选出满足除拍卖时间外所有条件的房源（单价<400元/m² + 精装修 + 出售 + 普通住宅 + 面积>=50㎡）
2. 找出筛选出的房源最早结束拍卖的房源
3. 获取结束时间 end_time_ms
4. 计算距离结束时间: delta = end_time_ms - now_ms
5.
   如果 delta > 30分钟:
     → 创建一次性任务在 (end_time - 30分钟) 执行

   如果 delta < 30分钟:
     → 创建一次性任务在 (当前时间 + 3分钟) 执行

```
#### 任务模板

创建新任务时，必须使用以下**完全一致**的描述：

```
【拍卖捡漏监控任务】

必须读取 ali-auction-deal-finder skill，然后严格按照 skill 执行。
```
## 判断标准

### 价格计算
```python
# 当前价格（元）= current_price_fen / 100
# 单价（元/平方米）= 当前价格 / 面积
```

### 装修等级判定
- ✅ 合格：豪华装修、精装修
- ❌ 不合格：普通装修、毛坯、简装、其他

### 剩余时间判定
- ✅ 捡漏机会：剩余时间 < 30分钟
- ⚠️  需关注：剩余时间 30分钟 ~ 1小时
- ⏰ 时间充足：剩余时间 > 1小时

## 输出格式
最终输出应包含以下信息：

```markdown
## 🔥 捡漏房源推荐

### 房源 1
- **标题**: xxx
- **小区**: xxx
- **面积**: xx m²
- **当前价格**: xxxx 元
- **单价**: xxx 元/m²
- **剩余时间**: xx 分钟
- **装修程度**: 精装修
- **链接**: [查看详情](xxx)
```

## 通知规则

- ✅ **满足所有条件**（单价<400 + 精装修 + 剩余时间<30分钟）→ **立即加急通知**
- ❌ **不满足条件** → **静默处理，不发送任何信息**


## 注意事项

1. **价格单位**: 系统中所有金额字段单位为"分"，1元 = 100分
2. **时间单位**: 时间戳为毫秒级 Unix 时间戳
3. **面积解析**: 面积可能以字符串形式返回（如"86.12 平方米"），需要提取数字
4. **装修字段**: 装修程度在标的物描述的 `decoration` 字段中
5. **时区设置**: 调度任务使用 Asia/Shanghai 时区
